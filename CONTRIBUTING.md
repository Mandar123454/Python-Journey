# Contributing to Python Journey

Thank you for your interest in contributing! Contributions are welcome for examples, fixes, docs, and improvements.

## How to Contribute
1. Fork the repo and create your branch from `main`.
2. Branch naming: `topic/short-description` (e.g., `gui/add-slider-example`).
3. Keep changes focused; avoid unrelated formatting changes.
4. Add or update README/docs where useful.
5. Run and verify examples you change.
6. Open a Pull Request with a clear description of what and why.

## Development Setup (Windows PowerShell)
```powershell
python -m venv .venv
. .\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt
```

## Code Style
- Prefer readable, educational code over clever one‑liners
- Keep files self‑contained and runnable
- Avoid adding heavyweight dependencies unless justified

## Commit Messages
- Use imperative mood: "Add X", "Fix Y"
- Reference files or topics when helpful

## Reporting Issues
- Include steps to reproduce
- Mention script names and expected vs actual behavior
- Propose a fix if you have one

Thanks for helping make this learning path better!
